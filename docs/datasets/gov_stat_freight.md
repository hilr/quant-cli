# gov_stat/freight — 货运量月度指标

| 项 | 内容 |
|---|------|
| 命令 | `uv run python -m quant.cli gov-stat-freight --data-path <readonly> --output-dir <dataset>` |
| 输入 | `{data_path}/gov_stats/货运量/{year}.xlsx` |
| 输出 | `{output_dir}/gov_stat/freight.csv`，长表，2005-01 起 |
| 逻辑 | 「指标×月份」宽表转长表，月份按列名 `YYYY年M月` 解析（列序倒序）；1-2 月合并缺口的当期值自动填补（见下） |

**字段：**
| 字段 | 含义 |
|------|------|
| date | 月份 YYYY-MM |
| indicator | 指标名（含单位后缀，见下） |
| value | 数值 |

**indicator 取值（20 个 = 5 主指标 × 4 子指标）：**
- 货运量（总）：当期值(万吨)、累计值(万吨)、同比增长(%)、累计增长(%)
- 铁路货运量：当期值、累计值、同比增长、累计增长
- 公路货运量：当期值、累计值、同比增长、累计增长
- 水运货运量：当期值、累计值、同比增长、累计增长
- 民航货运量：当期值、累计值、同比增长、累计增长

单位由 indicator 名的括号后缀标明：`万吨` 或 `%`。

## 1-2 月合并发布的三种模式

| 年份 | 模式 | 原始数据 | 填补规则 |
|---|---|---|---|
| 2005-2023 | SPLIT | 当期[1月]、当期[2月] 各自独立 | 不动 |
| 2024-2025 | MERGED | 当期[1月]、当期[2月] 都空；累计[2月] = 1+2 月合计 | **平摊**：Jan = Feb = 累计[2月] / 2 |
| 2026 | 半合并 | 当期[1月] 空，**当期[2月] 有值**；累计[2月] = 1+2 月合计 | **反推**：Jan = 累计[2月] − 当期[2月]，Feb 保留实际值 |

详见 [`gov_stat/port_freight`](gov_stat_port_freight.md) 同名段落。

## 源数据坑

- **月份列倒序**：表头月份从 12 月→1 月，按列名文本解析。
- **指标名空格**：xlsx 带尾随空格，已统一去除所有空白。
- **2026 文件不完整**：截至当前只到当年 5 月。
- **统计口径调整（影响同比可比性，跨年同比需谨慎）**：
  - **2014 起**：根据 2013 年交通运输业经济统计专项调查，公路水路运输量统计口径与推算方案调整，2014 月度数据按新方案推算。跨 2013/2014 的同比不可比。
  - **2020 起**：公路货物运输量、公路货物运输周转量按 2019 道路货物运输量专项调查调整后口径计算；水路运输（海洋）改企业联网直报，2020 增速按可比口径。跨 2019/2020 的同比不可比。
- **偶发当期值空缺**：2007-2018 年每年有 1 个非 1-2 月的当期值单元格漏填（多在 12 月），累计值完整但当前转换器不补 3-12 月空缺 → 该月当期值在输出中缺失（影响极小，约 12 个单元格）。

## 使用示例

```bash
uv run python -m quant.cli gov-stat-freight --data-path /mnt/readonly_dataset --output-dir /mnt/dataset
```
