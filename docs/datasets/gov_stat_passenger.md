# gov_stat/passenger — 客运量月度指标

| 项 | 内容 |
|---|------|
| 命令 | `uv run python -m quant.cli gov-stat-passenger --data-path <readonly> --output-dir <dataset>` |
| 输入 | `{data_path}/gov_stats/客运量/{year}.xlsx` |
| 输出 | `{output_dir}/gov_stat/passenger.csv`，长表，2005-01 起 |
| 逻辑 | 「指标×月份」宽表转长表，月份按列名 `YYYY年M月` 解析（列序倒序）；1-2 月合并缺口的当期值自动填补（见下） |

**字段：**
| 字段 | 含义 |
|------|------|
| date | 月份 YYYY-MM |
| indicator | 指标名（含单位后缀，见下） |
| value | 数值 |

**indicator 取值（20 个 = 5 主指标 × 4 子指标）：**
- 客运量（总）：当期值(万人)、累计值(万人)、同比增长(%)、累计增长(%)
- 铁路客运量：当期值、累计值、同比增长、累计增长
- 公路客运量：当期值、累计值、同比增长、累计增长
- 水运客运量：当期值、累计值、同比增长、累计增长
- 民航客运量：当期值、累计值、同比增长、累计增长

单位由 indicator 名的括号后缀标明：`万人` 或 `%`（**注意：客运量是「万人」，与货运量「万吨」不同**）。

## 1-2 月合并发布的三种模式

| 年份 | 模式 | 原始数据 | 填补规则 |
|---|---|---|---|
| 2005-2023 | SPLIT | 当期[1月]、当期[2月] 各自独立 | 不动 |
| 2024-2025 | MERGED | 当期[1月]、当期[2月] 都空；累计[2月] = 1+2 月合计 | **平摊**：Jan = Feb = 累计[2月] / 2 |
| 2026 | 半合并 | 当期[1月] 空，**当期[2月] 有值**；累计[2月] = 1+2 月合计 | **反推**：Jan = 累计[2月] − 当期[2月]，Feb 保留实际值 |

详见 [`gov_stat/port_freight`](gov_stat_port_freight.md) 同名段落。

## 源数据坑

- **月份列倒序**：表头月份从 12 月→1 月，按列名文本解析。
- **指标名空格**：xlsx 带尾随空格，已统一去除所有空白。
- **2026 文件不完整**：截至当前只到当年 5 月。
- **统计口径调整（影响同比可比性，跨年同比需谨慎）**：
  - **2013-2023**：公路客运量仅包含传统的班车包车客运量（不含城际公交、出租等），口径偏窄。
  - **2024 起**：公路营业性客运量口径大幅扩展，包含班车包车客运量、公共汽电车城际城乡客运量、出租汽车（含巡游出租、网约车）城际城乡客运量。**跨 2023/2024 的同比不可比**（公路客运量会出现数量级跳变）。
  - 总客运量 2020 年断崖下跌是新冠疫情所致（非口径问题）。
- **偶发当期值空缺**：2007-2018 年每年有 1 个非 1-2 月的当期值单元格漏填（多在 12 月），累计值完整但当前转换器不补 3-12 月空缺 → 该月当期值在输出中缺失（影响极小，约 12 个单元格）。

## 使用示例

```bash
uv run python -m quant.cli gov-stat-passenger --data-path /mnt/readonly_dataset --output-dir /mnt/dataset
```
