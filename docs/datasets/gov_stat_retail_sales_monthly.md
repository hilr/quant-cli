# gov_stat/retail_sales_monthly — 社会消费品零售总额每月新增额

| 项 | 内容 |
|---|------|
| 命令 | `uv run python -m quant.cli gov-stat-retail-monthly --data-path <readonly> --output-dir <dataset>` |
| 输入 | `{data_path}/gov_stats/社会消费品零售总额/{year}.csv`（≤2024）或 `.xlsx`（≥2025） |
| 输出 | `{output_dir}/gov_stat/retail_sales_monthly.csv`，宽表，2000-01 起 |
| 逻辑 | 取「累计值」指标，年内差分：每月新增额[t] = 累计值[t] − 累计值[t−1]，1 月 = 累计值[1月]。1-2 月合并缺口的 1 月累计值用「2 月累计值 / 2」平分填充 |

**字段（单位均为亿元，当月零售额）：**

| 字段 | 含义 |
|------|------|
| date | 月份 YYYY-MM |
| 总额 | 社会消费品零售总额当月值（= 当月新增额），2000-01 起 |
| 限上 | 限上单位消费品零售额当月值，2011-01 起（发布略滞后，可能比总额少尾月） |

## 算法

由累计值（YTD）年内差分得到当月零售额，而非直接取「当期值」列——因为 2012+ 的
1-2 月合并公布让当期值 1、2 月同时缺失，而累计值只缺 1 月，差分路径只需补 1 月一个缺口：

```
每月新增额[t] = 累计值[t] − 累计值[t−1]      # 2-12 月，年内差分（不跨年）
每月新增额[1月] = 累计值[1月]
累计值[1月] = 累计值[2月] / 2                 # 仅 2012+ 合并年份，1 月原缺失时
```

故合并年份 1、2 月新增额均 = 2 月累计值 / 2（合计平分，与官方累计值自洽）；2011 及
以前 1-2 月分开公布，1 月累计值原值直接用。已在 2010（分开）、2024（合并）与源数据对账一致。

## 源数据坑

- **1-2 月合并公布（2012 起）**：累计值只缺 1 月，用 2 月累计值 / 2 平分填充后再差分。
- **限上起步晚**：限上单位零售额 2011 起，之前为 null（132 个 null 全在 2011 前）。
- **限上发布滞后**：尾月限上可能比总额少 1 个月（如总额到 2026-04 时限上同步）。
- **指标名空格**：xlsx 带空格、csv 不带，已统一去除（复用 `_gov_stat_parse_indicator_table`）。
- **脚注过滤**：`注：`/`数据来源` 行已跳过。

## 使用示例

```bash
uv run python -m quant.cli gov-stat-retail-monthly --data-path /mnt/readonly_dataset --output-dir /mnt/dataset
```
