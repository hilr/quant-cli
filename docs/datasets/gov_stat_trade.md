# gov_stat/trade — 海关进出口月度指标

| 项 | 内容 |
|---|------|
| 命令 | `uv run python -m quant.cli gov-stat-trade --data-path <readonly> --output-dir <dataset>` |
| 输入 | `{data_path}/gov_stats/进出口/{year}.csv`（≤2022）或 `.xlsx`（≥2023） |
| 输出 | `{output_dir}/gov_stat/trade.csv`，长表，2000-01 起 |
| 逻辑 | 「指标×月份」宽表转长表，月份按列名 `YYYY年M月` 解析（列序乱序/倒序均可） |

**字段：**
| 字段 | 含义 |
|------|------|
| date | 月份 YYYY-MM |
| indicator | 指标名（含单位后缀，见下） |
| value | 数值 |

**indicator 取值（14 个）：**
- 进出口总值：当期值(千美元)、同比增长(%)、累计值(千美元)、累计增长(%)
- 出口总值：当期值、同比增长、累计值、累计增长
- 进口总值：当期值、同比增长、累计值、累计增长
- 进出口差额：当期值(千美元)、累计值(千美元)（无同比口径）

单位由 indicator 名的括号后缀标明：`千美元` 或 `%`。

## 源数据坑

- **月份列乱序**：csv 里月份列顺序是乱的（如 `2010年9月,2010年1月,2010年7月...`），xlsx 是倒序（12月在前）。按列名文本解析，不依赖列位置。
- **1-2 月合并公布（仅 2020 年）**：仅 2020 年因疫情统计局将 1-2 月合并发布，该年 1、2 月当期值为空（不生成行）；其余年份 1、2 月单独公布，当期值均有值。
- **格式过渡**：2023 起由 csv 改为 xlsx（前 2 行多出「数据库：月度数据」「时间：...」元数据，表头在第 3 行）。解析按「找含『指标』的行」自动适配。
- **指标名空格**：xlsx 指标名带空格（`当期值 (千美元)`），csv 不带（`当期值(千美元)`）。已统一去除所有空白。

## 使用示例

```bash
uv run python -m quant.cli gov-stat-trade --data-path /mnt/readonly_dataset --output-dir /mnt/dataset
```
