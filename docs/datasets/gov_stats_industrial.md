# gov_stats/工业企业指标 — 规模以上工业企业月度经济效益

国家统计局规模以上工业企业经济效益月度数据，按年分文件存放于 `/mnt/readonly_dataset/gov_stats/工业企业指标/`。每行为一个指标，每列为一个月份，值为累计值或期末存量。

**文件格式**

| 年份 | 格式 | 说明 |
|------|------|------|
| 2000–2024 | CSV | 第 1 行为表头 |
| 2025–2026 | XLSX | 前 2 行为元数据（"数据库：月度数据"、"时间：…"），第 3 行才是表头；指标名带空格（如 `利润总额累计值 (亿元) `） |

**使用前必读的特征**

1. **月份列顺序是乱的**：每年 12 个月份列顺序不固定（如 2020 年表头为 `3月,1月,9月,7月...`），必须解析列名提取月份再排序，**不能按列位置取值**。

2. **利润字段名跨年完全一致**：`利润总额累计值(亿元)`（字节级确认，无隐藏字符差异）。2024 年仅把该指标的行序从第 66 行挪到第 51 行（其它指标同步重排），字段名未变。

3. **每年 1 月恒为空**：统计局不单独公布 1 月，3 月才公布 1–2 月合并累计值。单月利润换算口径：1 月、2 月单月 = 2 月累计 ÷ 2；3–12 月单月 = 本月累计 − 上月累计。

4. **2007–2010 年只有季度数据（2/5/8/11 月）**：这四年所有有数据的字段都统一只在 2、5、8、11 月有值，没有任何字段月度更细，逐月单月利润无法完整计算。
   - 2007、2008、2009：相邻年份（2008/2009/2010 文件）的"上年同期累计值"列同样稀疏，**无反推路径**。
   - **2010 可反推**：2011 文件的"利润总额上年同期累计值"列为完整 2–12 月，等于 2010 同月累计，可借此重建 2010 完整月度（但为新口径，见下）。

5. **2011 年起口径调整**：规模以上工业企业门槛由"主营业务收入 500 万元"上调到"2000 万元"。后果：2011 文件的"上年同期"是**按新口径重述的 2010**，系统性偏低约 2–3%（4 个重叠月份无一吻合）；用其重建的 2010 为"新口径 2010"，与 2010 自身旧口径值不可混用。

6. **指标集随年份演变（多次断点，非一次性切换）**：各指标启用年份不同，跨年拼接需注意口径/字段缺失：

   | 字段 | 有数据的年份 |
   |------|------------|
   | 利润总额累计值 | 2000 至今（2007–2010 仅 2/5/8/11 月） |
   | 主营业务收入累计值 | 2000–2018（2019 起被"营业收入"取代） |
   | 营业收入累计值 | 2019 至今 |
   | 营业利润累计值 | 2012/2013 残缺、2018 起完整 |
   | 存货本月末 | 2010 至今 |
   | 所有者权益合计 / 平均用工人数 | 2020 起 |
   | 利息费用累计值 / 应收票据及应收账款 | 仅 2019（过渡年，其余年份空） |

7. **2026 年残缺**：当前数据快照下仅有 2、3、4 月数据，5 月尚未公布。

**2007–2009 年字段清单**（三年结构一致，共 76 个字段）

有数据的字段（均只 2/5/8/11 月，每个含本期 / 上年同期 / 增减% 三条）：
- 规模：企业单位数、亏损企业数
- 资产负债（期末存量）：流动资产合计、应收账款、产成品存货、资产总计、负债合计
- 损益（累计值）：主营业务收入、主营业务成本、主营业务税金及附加、销售费用、管理费用、财务费用、利息支出、**利润总额**、亏损企业亏损总额、应交增值税

整列为空的字段：应收票据及应收账款、存货、所有者权益合计、营业收入累计值、营业成本累计值、利息费用累计值、投资收益累计值、营业利润累计值、平均用工人数。
