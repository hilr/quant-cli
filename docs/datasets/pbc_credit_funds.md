# pbc/credit_funds — 金融机构信贷收支表（存贷款）

| 项 | 内容 |
|---|------|
| 命令 | `uv run python -m quant.cli pbc-credit-funds --data-path <readonly> --output-dir <dataset>` |
| 输入 | `{data_path}/gov_pbc/{year}/[金融机构信贷收支统计/]金融机构{本外币,人民币}信贷收支表.{htm,xls,xlsx}` |
| 输出 | `{output_dir}/pbc/credit_funds.csv`，长表，1999-01 起 |
| 格式优先级 | xlsx > xls > htm |
| 单位 | 亿元 |

**字段：**
| 字段 | 含义 |
|------|------|
| date | 月份 YYYY-MM |
| currency | `本外币` 或 `人民币`（口径） |
| item | 项目名（来源方/运用方全明细，带层级上下文，见下） |
| value | 月末余额（亿元） |

**item 命名规则**：2015+ 源表为多级层次，同一叶子名会在不同父项下重复（如「（1）活期存款」同时挂在住户存款、非金融企业存款下；「消费贷款」同时挂在短期贷款、中长期贷款下）。为保证 `(date, currency, item)` 唯一，叶子项用「·」拼接父级上下文：

- 分组子项（`（N）xxx`）补所属分组：`住户存款·（1）活期存款`、`非金融企业存款·（1）活期存款`、`企（事）业单位贷款·（2）中长期贷款`
- 裸叶子再叠一层中间项：`住户贷款·短期贷款·消费贷款`、`住户贷款·中长期贷款·经营贷款`
- 顶级行**保持原名**（绝不补父级），靠前缀模式识别：
  - 章节标题 `一、/二、…`（被规范化剥掉前缀，如 `一、各项存款` → `各项存款`）
  - 子段 `（一）/（二）…`：`（一）境内存款`、`（二）境外贷款`
  - 分组父项 `1./2./3.…`：`1.住户存款`、`2.非金融企业存款`、`3.机关团体存款`、`4.财政性存款`、`5.非银行业金融机构存款`、`2.企（事）业单位贷款` … — **这些是兄弟关系，不是父子关系**，不能因前一行是「1.住户存款」就把「2.非金融企业存款」当成它的子项（早期实现犯过这个错，已修）

**item 包括**（按出现顺序，含汇总行和分项）：来源方——各项存款（含住户存款、非金融企业存款、机关团体存款、财政性存款、非银行业金融机构存款等分项）、金融债券、对国际金融机构负债、其他、资金来源总计；运用方——各项贷款（含住户贷款、企（事）业单位贷款、非银行业金融机构贷款等分项，再分短期/中长期/票据融资/融资租赁等）、债券投资、股权及其他投资、在国际金融机构资产、资金运用总计。

## 源数据坑

- **本外币口径部分年份缺失**：1999-2003 等早期年份只有人民币表，无本外币表，对应行不生成（正常）。
- **「（按部门）」表排除**：原始数据另有「金融机构人民币信贷收支表（按部门）」，本数据集不用（按 exclude 过滤）。
- **全明细输出**：约 30-50 个项目行/月，含汇总行（各项存款/各项贷款/资金来源总计/资金运用总计）和所有分项。项目名已去英文翻译、去中文序号前缀（「一、」）、折叠缩进空白。
- **跨分组重名消歧**：叶子项补父级上下文（见上「item 命名规则」），消除 `（1）活期存款`/`消费贷款`/`保证金存款` 等跨分组重名导致的重复。注意早期（如 2010）源表已自带限定名（`（1）活期储蓄`/`（1）活期存款` 不重名），无需补全。
- **层级缩进**：分项用空格/全角空格缩进表示层级，但缩进在 htm/xls 中不一致、不可靠，消歧改用序号前缀（`1.`/`（1）`）推断层级。

## 使用示例
